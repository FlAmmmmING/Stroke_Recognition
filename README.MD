2024/1/23 蒋玮杰
    目前提取了骨架中可能的起始点和交点，目前的思路是通过如下的数据复现出书法文字的书写过程
        数据1：这个书法字的笔画
        数据2：这个书法字的起始点和交点
        数据3：这个书法字的骨架
    目前的难点：
        1.如何判定这个笔画的开始和结束？——开始和结束一定是某一个交点或者是起始点
        2.如何判定这个笔画的方向？——如果一个平行于x轴的笔画骨架，那它有可能是横，也有可能是从右到左的撇
        3.如何判定一些特殊的笔画？——比如横折钩这种？
    对于难点的思路：
        1.遍历图片的像素点的顺序很重要，应该采取从左到右从上到下的规则，这是因为汉字的书写顺序往往遵循这种规则
            为什么不是先遍历从上到下再遍历从左到右？其实差不多，但是前者的遍历方便我们识别“撇”
        2.如何判定什么时候需要合并交点？
            可以去计算图片的大小，然后按照一定比例规定范围，这是一个比较粗略的算法。
            或者可以计算所有笔画长短然后算一个阈值，其实思想和上一种一样。
            总感觉这种方法真的简陋……
        3.如何遍历？
            分为三种：
                1.如果这个点是起始点，那没什么问题，因为起始点只有一条路和它相连，只需要沿着走就行了
                2.如果这个点是路径，总体上和起始点一样，沿着一个没有走过的地方遍历，然后直到找到另一个起始点/交点然后进行后续判断
                3.如果这个点是交点，就需要判断周围是否有过于近的交点，看看是不是能合并，如果能合并，就朝着目标交点遍历。
                4.关于笔画遇到交点是否需要延伸这个问题：如果遇到交点之后，笔画还能够按照既定的方向移动，就这么延伸下去，反之则停下。    
    今天就想到了这么多，先做起来吧……
